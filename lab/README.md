# Lab

用于进行相关可行性实验。

## 命名

- 实验目录使用 `00-xxx`、`01-xxx` 形式命名。
- 命名保持高度概括与简略。

## 结构

- 每个实验使用一个独立文件夹。
- 每个实验文件夹内包含测试脚本、详细记录和本地产物。
- 仅在确有复用价值时，才放入 `shared/` 共享。
- 当前已有 `05-数据源可达性` 用于分析数据源不可达层级。
- 当前已有 `06-接口对照` 用于确认 AkShare 请求接口是否正确。
- 当前已有 `07-直连采集` 用于验证 httpx 直连东财接口是否可稳定采集。
- 当前已有 `08-最小策略闭环` 到 `12-风险指标闭环` 用于策略、回测、成本、组合和风险闭环。
- 当前已有 `13-调仓执行闭环` 到 `17-结果持久化闭环` 用于执行、滑点、约束、调度和持久化闭环。
- 当前已有 `18-数据一致性与污染` 到 `22-调仓频率与成本` 用于一致性、时间对齐、部分成交、数值稳定性和频率成本闭环。
- 当前已有 `23-复权一致性与停牌处理` 到 `30-事件回放与幂等性` 用于复权、动态 universe、稳健协方差、优化回退、T+1、风险状态机、触发执行和回放幂等闭环。
- 当前已有 `31-策略层解耦` 到 `38-数据版本锁定与回放` 用于策略纯函数、账户状态、订单生命周期、事件驱动、多策略分配、分层风控和版本锁定闭环。
- 当前已有 `39-模块替换与接口稳定性` 到 `43-横向扩展与负载增长` 用于模块替换、执行环境切换、时间尺度切换、故障注入和横向扩展闭环。
- 当前已有 `44-增量同步与Parquet缓存` 到 `46-入口隔离与标准化` 用于增量同步、缓存、收盘决策和入口隔离标准化闭环。

## 记录

- 单个实验的过程与步骤记录写入各自目录下的 `record.md`。
- 每个实验结束后，将高度概括的结论汇总到 `records.md`。
