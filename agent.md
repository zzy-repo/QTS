# Agent

## 写作原则

1. `agent.md` 的书写必须严格遵从渐进式披露。
2. 仅保留关键约束、关键入口与必要说明，其余内容交由 agent 自行查看相关文件。
3. 避免一次性展开大量背景、流程或实现细节。

## 文档约束

1. 相关文档不得超出用户明确提示的观点范围。
2. 不补充未经要求的延伸判断、方案设想或额外主张。
3. 表达尽量简略，并保持高度概括。

## Lab 约定

1. `lab/` 文件夹仅用于相关可行性实验。
2. 单个实验的过程与步骤单独记录。
3. 单个实验结束后，将高度概括的结论汇总到 `lab` 下统一实验记录中。
4. 实验目录使用 `00-xxx`、`01-xxx` 形式命名，目录内以 `run.py` 作为入口。
5. 命名要求高度概括且简略。
6. 如果不能确保代码逻辑成功运行，必须先在 `lab/` 做完整可行性实验确认工程可行性，再写正式代码；不要先写代码再事后验证。

## 正式系统

1. `qts/` 是正式主代码，`lab/` 只能作为验证层，不得被 `qts/` 反向导入。
2. 正式系统的补充文档写在 `doc/` 下，并且要跟 `qts/` 的文件保持同步。
3. `doc/` 里的内容只做每个 `qts` 文件的总括，不展开实验细节。

## 输出原则

1. 代码执行后的输出统一使用中文。
